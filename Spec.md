# ToDoアプリ 仕様書

## 1. 概要
シンプルで使いやすいToDo管理アプリケーション。
ユーザーがタスクを追加、一覧表示、完了状態の切り替え、削除ができる。

## 2. 機能要件
- **タスク一覧表示**: 登録されているすべてのタスクを表示する。
- **タスク追加**: 新しいタスクを入力してリストに追加する。
- **タスク完了/未完了の切り替え**: チェックボックス等でタスクの完了状態を変更できる。
- **タスク削除**: 不要なタスクをリストから削除できる。
- **データの永続化 (任意)**: ブラウザの `localStorage` を使用して、リロード後もデータを保持する。

## 3. UI/UX デザイン案
- **ヘッダー**: アプリのタイトル「ToDo App」。
- **入力エリア**: テキスト入力フィールドと「追加」ボタン。
- **リストエリア**:
    - 各タスクを行として表示。
    - 左側に完了チェックボックス。
    - 中央にタスク内容（完了済みは打ち消し線を表示）。
    - 右側に「削除」ボタン。
- **レスポンシブ対応**: モバイルとデスクトップ両方で快適に操作できる。

## 4. 技術スタック
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS
- **State Management**: React `useState`, `useEffect`
- **Language**: TypeScript

## 5. 実装ステップ
1. 基本的なレイアウトの作成。
2. タスク追加機能の実装。
3. タスク一覧表示と削除機能の実装。
4. 完了状態の切り替え機能の実装。
5. `localStorage` による永続化の追加。
